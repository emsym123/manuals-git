
git workflow使用文档
==================================

从Master创建一个分支，写代码，然后将这个分支合并到Master，这个代码合并要先 **code  review** 和 **test** ,这样才能保证代码的质量。   
   
在图中 
 ``master`` 作为主干，表示代码已发布给客户。  
  
 ``release branch``   是未发布准备发布的代码。   
 
在平时工作中，作为开发人员是对应着 ``develop``  工作、更新
